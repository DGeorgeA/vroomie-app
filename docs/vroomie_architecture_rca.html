Part 2 — Root Cause Analysis

⚠️ P0 — Production Blocker: Customers reported zero anomaly detections across all fault types.

2.1 Incident Summary

Four independent bugs combined to create a situation where no real-world anomaly could ever be detected. The most critical was a circular dependency deadlock in the ML engine initialization.

2.2 Root Cause Matrix

#BugSeverityFile
1Circular init deadlock — fingerprints never loadCRITICALmlEmbeddingEngine.js
2Classification cadence never fires (modulo arithmetic)HIGHaudioFeatureExtractor.js
3False silence rejections (block-level RMS)HIGHaudioFeatureExtractor.js
4Threshold impossibly strict (0.90 cosine)HIGHmlEmbeddingEngine.js

2.3 Bug 1 — Circular Initialization Deadlock (PRIMARY)

This is the smoking gun. Without reference fingerprints, findBestMatch() always returns {status: 'normal', reason: 'no_references'} regardless of audio input.

Call Stack (Before Fix)

initializeEmbeddingEngine()
  → initializeFingerprints()           // Called FIRST
    → loadOrGenerateFingerprints(getAudioEmbedding)
      → [no cache] → fetch WAVs → getAudioEmbeddingFn(pcm)
        → getAudioEmbedding(pcm)       // yamnetModel is NULL!
          → initializeEmbeddingEngine() // RECURSIVE!
            → initializeFingerprints()  // Still empty!
              → loadOrGenerateFingerprints(...) // ∞ LOOP

Impact

  • New users: Total detection failure — fingerprints never generate, every classification returns "no_references"
  • Existing users with cache: May work if a previous version successfully cached fingerprints
  • Users who clear browser data: Detection breaks permanently until next code fix

Fix Applied

// BEFORE (broken):
async function initializeEmbeddingEngine() {
  await initializeFingerprints();  // ← needs model!
  // ... load model (too late) ...
}

// AFTER (fixed):
async function initializeEmbeddingEngine() {
  await loadYamnetModel();         // ← model FIRST
  // Model now works, so getAudioEmbedding() works
  fingerprints = await loadOrGenerateFingerprints(getAudioEmbedding);
}

2.4 Bug 2 — Classification Never Fires

// BEFORE
if (totalSamples % (SCRIPT_BUFFER_SIZE * 2) === 0) // Fragile modulo

// AFTER
if (performance.now() - lastClassifyTime >= 900)    // Wall-clock timing

The modulo check required totalSamples to land exactly on a multiple of 8192. Async YAMNet calls caused sample counts to skip past boundaries.

2.5 Bug 3 — False Silence Rejections

// BEFORE — RMS over 256ms block (misses impulsive sounds)
const rms = Math.sqrt(frameRmsSq / blockSize);

// AFTER — RMS over full 1-second snapshot
for (let i = 0; i < windowSamples; i++) {
  snapshotRmsSq += snapshot[i] * snapshot[i];
}
const rms = Math.sqrt(snapshotRmsSq / windowSamples);

2.6 Bug 4 — Impossibly Strict Threshold

// BEFORE
const ANOMALY_THRESHOLD = 0.90; // Real-world scores: 0.70-0.85

// AFTER
const ANOMALY_THRESHOLD = 0.75; // Accounts for mic noise, reverb, distance

2.9 Prevention Recommendations

  1. Startup self-test: Verify yamnetFingerprints.length === 6 on init — log FATAL if not
  2. Integration test: Play each reference WAV through the pipeline and verify detection
  3. Never set thresholds above 0.85 for microphone-based audio matching
  4. Never put initialization dependencies in circular order
  5. Always compute RMS over the classification window, not individual blocks
